Best Practice - class vs id Explained

25 April 2020


HTML elements like div , p , span create the structure of a webpage. Think of a house and the rooms in it like the kitchen, bedrooms and so on. Each element is like a different room in the house.

CSS can be used to change the look of an element. Think of CSS as the interior design and style of each room in the house. In order to customise an HTML element using CSS, you need to target the element that you want to change.

One of the ways for CSS to target elements is through the use of class and id.


What are class and id?

Class and id are labels or identifiers that you can give to HTML elements.

You assign a class or id to an element so that CSS can target the correct element that you want to change. When the correct element is targeted you can then specify how you want that element to look and behave.

You can assign both class and id to an element at the same time, too.


What's the difference between class and id?

The differences between class and id are:

  • class can be applied multiple times on a page to multiple elements. An element can also be given more than one class.

    Going back to the analogy of the interior design of a house, using class to target an element is like deciding on a thematic style such as shabby chic. The same style can be applied to different rooms in the house.

  • id can only be used once in the whole page. An element can only have one id.

    Think of using id as the interior designer identifying a statement piece in the room. It's the one thing that is different to the thematic style of a room and isn't found elsewhere in the house.


When do I use class and id?

Now that the differences between class and id have been established, when trying to decide whether to assign class or id to an element, ask yourself this question:

“Am I likely to repeat the same custom style anywhere else on the page?”

If the answer is even a slight maybe, then use class.

For id, the general rule of thumb is to only use it for a style that you will not use anywhere else on the page.

In other words, if there’s something on your website that needs to look different to everything else or isn’t repeated elsewhere then you’d give it an id in order to change its look.

It’s worth noting also that id is treated as more important than class so it will override any custom properties defined by a class. For example, if in a div container there’s an element that needs to look different to everything else in it then you can assign an id to this particular element.
